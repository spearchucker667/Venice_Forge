### 2026-07-20 — Phase 2.6 folder-backup encryption rewrite (libsodium)

- **Service body rewrite (`electron/services/chatFolderBackupService.ts`):** replaced the prior `crypto.randomBytes().toString("base64")`-as-passphrase pattern (which stored the random IV in the `keyWrapped` field) with a libsodium Argon2id13 + XChaCha20-Poly1305-IETF envelope. Per backup: 256-bit fresh DEK (`crypto_aead_xchacha20poly1305_ietf_KEYBYTES`), 16-byte salt, Argon2id-derived KEK at `_OPSLIMIT_INTERACTIVE` / `_MEMLIMIT_INTERACTIVE`, KEK nonce, and an independent payload nonce. Wire format v2 ships `{ version: 2, exportedAt, kdf: { algorithm: "argon2id13", opslimit, memlimit }, crypto: { salt, kekNonce, wrappedKey, payloadNonce, ciphertext }, publicHeader: { sourceFolderKind, sourceFolderName, createdAt, appVersion, includesMedia, conversationCount } }`. The clear-text manifest never appears on disk; conversations, messages, and folder metadata live exclusively inside the encrypted payload while `publicHeader` carries only the labels needed for passwordless preview.

The import path was already correct because opslimit/memlimit travel with the envelope, so `_sodiumReadyPromise` threaded through the import call-path resolves the rest.
