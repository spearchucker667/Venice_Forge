# Venice Forge Implementation TODO

## Phase 1: Security & Safety (Severity: Critical/High)
*   **[BUG-001]** Jina API Safety Guard Bypass: `electron/ipc/handlers.ts` and `src/research/providers/jinaResearchProvider.ts`. The Jina search and scrape requests (`jina:request` and `jinaFetch`) completely bypass the `assessChildExploitationSafety` guard. Search queries can contain CSAM keywords and will be blindly forwarded to the Jina API. Remediation: Enforce the `assessChildExploitationSafety` and `recordDecision` functions on Jina queries and payloads before making Jina network requests (both in the IPC handler and the web proxy layer).
*   **[BUG-002]** SSRF Bypass via Non-Canonical IPv6: `electron/utils/urlSecurity.ts`. The `isPrivateHostname` function fails to block IPv4-compatible IPv6 addresses (e.g., `::127.0.0.1`) and compressed IPv6 loopback variants (e.g., `0::1`, `0000::1`), allowing attackers to bypass the SSRF filter and make local requests. Remediation: Expand IPv6 parsing to handle full canonicalization of zero-compression and IPv4-mapped/compatible addresses.

## Phase 2: Architecture & Storage (Severity: High/Medium)
*   **[BUG-003]** Web Mode Jina API Key Stored in localStorage: `src/services/desktopBridge.ts`. The `desktopJinaApiKey.set` function writes the Jina API key directly to the renderer's browser `localStorage` in web mode, directly violating the repository rule against storing secrets in the renderer. Remediation: Implement a proxy route in `server.ts` for Jina requests and configure web mode to read the API key strictly from the server's `.env` file, matching the Venice API architecture.
*   **[BUG-004]** Markdown Copy Button Copies HTML-Escaped Code: `src/utils/markdown.tsx`. The raw `code` block text is HTML-escaped early in `minimalMarkdown`, which means the `data-code` payload for the copy button contains HTML entities (`&lt;`, `&gt;`). Users copying the code block receive broken HTML-escaped strings. Remediation: Base64-encode the raw, unescaped `code` string for the `data-code` attribute before it is processed by the global `escapeHtml` function.
*   **[BUG-005]** Markdown Parser Crash via Unpaired Surrogates: `src/utils/markdown.tsx`. Calling `encodeURIComponent` on a code block containing unpaired surrogates throws a `URIError`, which causes an unhandled React crash (Denial of Service). Remediation: Add a safe wrapper that strips or replaces unpaired surrogate characters before running `encodeURIComponent`.

## Phase 3: Packaging & Resilience (Severity: Medium/Low)
*   **[BUG-006]** Release Checksum Script Omitted from Dist Commands: `package.json`. The `dist:*` scripts do not invoke `npm run checksum:release`, causing the required `.sha256` sidecar verification in `npm run verify:dist` to fail after a clean build. Remediation: Add `&& npm run checksum:release` to the `dist`, `dist:win`, `dist:portable`, and all `dist:mac` npm scripts to ensure verification artifacts are generated alongside the binaries.
